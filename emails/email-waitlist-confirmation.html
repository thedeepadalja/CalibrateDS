More Framework Support
CalibrateDS currently verifies design-to-code fidelity for the frameworks it supports today. Expanding this means ptb scan and ptb stamp working natively against Vue, Svelte, Angular, and React Native — not just React.

Design Tools
◈
More Design Tool Integrations
Right now CalibrateDS is built around Figma. But design systems don't all live in Figma. This feature brings the same governance layer — variable verification, component stress testing, design drift detection — to Penpot, Sketch, and wherever else your source of truth lives.
→ If you're on this list: tell me which tool your team uses and why you moved away from Figma (or never used it).
GitHub Integration
⌥
GitHub Integration
CalibrateDS as a CI/CD gate. Every PR gets a design fidelity check — ptb scan runs on push, pixel diffs surface in the PR review, and the build fails if components have drifted from their Figma source. Design drift becomes a blocking condition, not a post-merge discovery.
→ If you're on this list: you're thinking about team adoption, not just personal use. Reply and tell me how your current design-to-code review process works (or doesn't).
